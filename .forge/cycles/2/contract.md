# Cycle 2 Contract — `state-persistence-and-recovery`

## Behavior

After this cycle, the `start` MCP tool persists and recovers per-project session state at `<projectRoot>/.sui-deepbook-course/state.json`. An absent file is a clean first run (no error, no warning). A valid file with the expected `schema_version` is loaded and surfaced under `start.state` so cycle 4 can resume at the recorded cursor (AC-7.1, AC-7.4). All state writes go through a write-tmp-then-fsync-then-rename adapter; a process killed mid-write can never leave a partially-written `state.json` behind — the previous good copy survives because `rename` is the only mutation of the canonical filename. A JSON-parse-failure on an existing file is treated as **corruption**: the original bytes are archived to `<projectRoot>/.sui-deepbook-course/state.corrupt-<ISO-timestamp>.json`, the canonical file is left untouched (the archive is the rescue path; the student decides on resume-from-zero vs abort in cycle 4), and `start` returns a structured `state-corrupt` warning naming the archive path (AC-7.2). A JSON-parseable file whose `schema_version` is unknown is treated as a **schema-mismatch guided stop**: no archive is written, no fresh state is silently created, and `start` surfaces a `state-schema-mismatch` warning carrying the offending version and a manual-migration message (AC-7.3). Corruption and schema mismatch are deliberately distinct response kinds — the contract refuses to conflate them.

The `start` tool keeps cycle 1's overall response shape; the only field that changes is `state`, which is now `State | null` instead of always `null`. Cycle 1's AC-1.3 invariant is preserved: when `outputStyleOk === false`, `start` performs zero filesystem writes (and zero filesystem reads beyond the registry + settings probes — in particular, no state archive is created, no `.sui-deepbook-course/` directory is created). Cycle 2 also extends `scripts/e2e/harness.ts` to drive the new state scenarios (E-006, E-015, E-016) and addresses cycle 1's carry-forward C018 by replacing the direct `runStart` import with a real MCP client connected over an in-memory transport — so the harness exercises the JSON-RPC framing the spec's `## MCP Tools` section assumes, and adding cycle 3's `runPreflightProbe` next cycle will not require touching a tool-name `if`-branch.

## Files

- `mcp/server/src/state.ts` — Load, save, archive, and schema-gate logic for `<projectRoot>/.sui-deepbook-course/state.json`. Exports `loadState(projectRoot) → { kind: 'absent' | 'ok' | 'corrupt' | 'schema-mismatch', state?, archivedTo?, foundVersion?, message }` and `saveState(projectRoot, state) → Promise<void>` (atomic write-tmp-then-fsync-then-rename). Owns the `STATE_SCHEMA_VERSION` constant (initial value `1`, matching spec.md `## State Schema`). Does **not** know about `start`'s response shape — pure I/O + classification.
- `mcp/server/src/schemas/state.ts` — Type definition + runtime validator for the `State` shape from spec.md `## State Schema` (`schema_version`, `selected_path`, `personalization`, `cursor`, `ladder`, `history`). Mirrors the structure of `schemas/path.ts` and `schemas/phases.ts` (cycle 1's pattern); branch-on-`err.code` style, no `existsSync` precheck, no bare `catch` (lessons from C002/C004/C010). Spot-content rules and ladder-rung gating remain cycle 4/5's concern; cycle 2 only enforces top-level shape and `schema_version`.
- `mcp/server/src/tools/start.ts` — Extended to call `loadState(projectRoot)` after the existing registry + output-style probes. Maps the four `loadState` outcomes onto the response: `absent` → `state: null`, no warning; `ok` → `state: <State>`, no state-warning; `corrupt` → `state: null` + push `state-corrupt` warning carrying the archive path; `schema-mismatch` → `state: null` + push `state-schema-mismatch` warning carrying the offending version. Preserves the AC-1.3 zero-write invariant: when `outputStyleOk === false`, `loadState` is **not** invoked at all (early return before any state probe). The handler does not call `saveState` in cycle 2 — no caller needs to write yet; the save adapter is exercised exclusively by unit tests this cycle and by cycle 4's `selectPath`/`setPersonalization`/`nextSpot` callers.
- `mcp/server/src/registry.ts` — **Unchanged** in shape; cycle 1's `RegistryWarning` union may need to widen to accept `state-corrupt` and `state-schema-mismatch` if `start.warnings` continues to be a single flat list. Implementer's choice: either widen the existing union, or split state warnings into a separate `start.stateWarning?: StateWarning` slot. Either way, the on-the-wire response stays additive (no field renames, no removals).
- `mcp/server/src/index.ts` — Export a `registerTools(server)` factory so the in-process harness can mount the tool set on an in-memory transport without spawning a child process. The bin entrypoint shrinks to the four lines that construct the server, call `registerTools`, and connect stdio. Resolves cycle 1 carry-forward C018+C020.
- `scripts/e2e/harness.ts` — Rewritten to construct an `McpServer` via `registerTools`, connect it to an in-memory `InMemoryTransport` pair from `@modelcontextprotocol/sdk`, and use the SDK's `Client` to call tools. Removes the `if (toolName === 'start')` branch — every registered tool is callable through the same surface. Adds fixture helpers `withCorruptState(projectRoot, bytes)`, `withValidState(projectRoot, state)`, and `withFutureSchemaState(projectRoot, version)` so cycle-2 scenarios (and cycle 4+'s) can stage `state.json` deterministically.
- `tests/state.test.ts` — Unit tests for `state.ts`: absent → first-run; valid → loaded; corrupt JSON → archive + classification; valid JSON with unknown `schema_version` → schema-mismatch (and **no archive written**); atomic save invariant proven by mocking `fs.rename` to throw and asserting the canonical file is unchanged; concurrent-corruption stress (two corruptions in the same second) produces two distinct archive filenames; the AC-1.3-adjacent invariant that `loadState` itself does not create the `.sui-deepbook-course/` directory unless an archive is being written.
- `tests/start.tool.test.ts` — Extended (or split into `tests/start.state.test.ts`) to assert: state-absent path returns `state: null` with no state-warning; state-ok path returns the loaded state; state-corrupt path returns `state: null` + a `state-corrupt` warning naming the archive; state-schema-mismatch path returns `state: null` + a distinct `state-schema-mismatch` warning that does **not** name an archive (no archive was written); AC-1.3 preserved when `outputStyleOk === false` — no archive, no `.sui-deepbook-course/` mkdir, no read of `state.json` under any of the four classifications.
- `tests/harness.mcp.test.ts` — Boots the harness through the new MCP transport seam and asserts a real JSON-RPC `tools/call` round-trip for `start`, replacing the synthetic envelope T-028 was building in cycle 1. Drives the three new state scenarios end-to-end through that surface: E-006 (corrupt → archive + warning), E-015 (valid state → resume cursor), E-016 (future schema → guided-stop warning, no archive).
- `tests/fixtures/state/valid-cursor-p2.json` — Fixture: schema-valid, `schema_version: 1`, cursor at `p2-retry/p2-spot-1`, `ladder.p1-spot-1.hint_used: true`. Used by E-015 + unit tests.
- `tests/fixtures/state/corrupt.json` — Fixture: truncated/non-JSON bytes. Used by E-006 + unit tests.
- `tests/fixtures/state/future-schema.json` — Fixture: JSON-valid, `schema_version: 999`, otherwise well-shaped. Used by E-016 + unit tests.
- `mcp/server/package.json` — Add `@modelcontextprotocol/sdk` client + in-memory-transport imports if they are not already covered by the cycle 1 dep entry; no new third-party deps beyond the SDK.

## Acceptance

- A1. `mcp/server/src/state.ts` exports `loadState(projectRoot)` and `saveState(projectRoot, state)`; `tsc --noEmit` is green under cycle 1's `strict: true` + `moduleResolution: "NodeNext"` config.
- A2. `loadState` on a project root without a `.sui-deepbook-course/` directory returns `{ kind: 'absent' }` and creates **zero** filesystem entries (no implicit `mkdir`, no canary file). Proven by spying on `fs.mkdir`/`fs.mkdirSync` and asserting zero invocations under this path. (AC-7.4)
- A3. `loadState` on a project root with a JSON-valid `state.json` whose `schema_version` matches `STATE_SCHEMA_VERSION` and whose top-level shape passes the schema validator returns `{ kind: 'ok', state }` carrying the parsed `State`. Cursor, personalization, and ladder fields round-trip exactly. (AC-7.1)
- A4. `loadState` on a project root with a `state.json` whose bytes are not valid JSON archives the original bytes byte-for-byte to `<projectRoot>/.sui-deepbook-course/state.corrupt-<ISO-timestamp>.json` and returns `{ kind: 'corrupt', archivedTo, message }`. The canonical `state.json` is left in place (so a subsequent `start` call still classifies as `corrupt` until the student or cycle 4's recovery flow rewrites it); the archive uses an ISO-8601 timestamp suffix and two corruptions in the same wall-clock second produce two distinct archive paths (e.g. via millisecond precision or a monotonic counter — implementer's choice, but the property is asserted in tests). (AC-7.2)
- A5. `loadState` on a project root with a JSON-valid `state.json` whose `schema_version` is unknown (e.g. `999`) returns `{ kind: 'schema-mismatch', foundVersion, message }`. **No archive file is created** under this path; the canonical `state.json` is not modified; `state.corrupt-*` files do not appear. The classification is structurally distinct from `corrupt` — different `kind` value, different message wording, no `archivedTo` field. (AC-7.3, plus the cycle's load-bearing "do not conflate" promise.)
- A6. `loadState` distinguishes `ENOENT` (kind `absent`) from `EACCES`/`ENOTDIR`/other `readFile` errors (kind `corrupt` with a wrapped `read-error` message). No bare `catch {}` in `state.ts`; every catch binds the error and branches on `err.code`. Reviewer "error-handling" dimension grep target. (Resolves cycle 1 C002/C010 lesson on the new surface.)
- A7. `saveState` writes via the sequence `writeFile(tmpPath, bytes)` → `fsync(fd)` (or `fs.fsyncSync` on the open tmp file) → `rename(tmpPath, statePath)`. Tests prove the atomicity invariant by mocking `fs.rename` to throw before invocation and asserting that the canonical `state.json` retains its prior contents (or remains absent if there was no prior file). The tmp file lives under `.sui-deepbook-course/` (same filesystem as the target so `rename` is atomic on POSIX); a unique tmp suffix prevents collision under concurrent saves.
- A8. `saveState` round-trips: `saveState(root, s)` followed by `loadState(root)` returns `{ kind: 'ok', state: s }` for any `State` that satisfies the schema validator.
- A9. `tools/start.ts` invokes `loadState` only after `outputStyleOk` is `true`. When `outputStyleOk === false`, `loadState` is **not** called and no entries are created under `.sui-deepbook-course/`. Reviewer "correctness" target: spy on `loadState`, `fs.mkdir*`, `fs.writeFile*`, `fs.rename*`, `fs.openSync` (write-flag filter) and assert zero invocations under the false-output-style fixture. (Preserves AC-1.3.)
- A10. `start`'s response shape is additive vs cycle 1: `state` widens from `null` to `State | null`; warnings are extended to carry `state-corrupt` and `state-schema-mismatch` kinds with structurally distinct payloads (one names an archive path, the other names an offending version). No existing field is renamed or removed.
