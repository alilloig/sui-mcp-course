# Phase 3 — Polling Loop with Failure Tolerance

In this phase you build the React polling loop that continuously fetches the order book state and displays it, while tolerating occasional failures without flickering the UI with transient errors.

## What you build

A `useEffect` hook at lines 116–145 of `src/App.tsx` that:

1. Calls `midPrice` and `getLevel2TicksFromMid` (both wrapped in `withRetry`) on every tick.
2. Polls at the personalized interval — the `{{ poll_interval_ms }}` placeholder will be substituted with your chosen value (e.g. `3000` for 3 seconds) at course-runtime.
3. Tracks `consecutiveFailures` and surfaces the error to the UI **only after 3 consecutive failures** — single transient errors are swallowed.
4. Cleans up the interval on component unmount.

## Key concepts

- **Consecutive-failure counter**: a variable defined outside the async tick callback, reset to 0 on every success, incremented on every failure.
- **Error threshold**: `if (consecutiveFailures >= 3)` gates the `setError` call so one bad tick does not flash an error banner.
- **Immediate first tick**: call `tick()` once synchronously before the interval starts so there is no blank screen on mount.
- **Cleanup**: return `() => clearInterval(intervalId)` from `useEffect` to avoid memory leaks.
