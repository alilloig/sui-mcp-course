# Phase 3 — Polling Loop with Failure Tolerance

In this phase you wire up the live polling loop that continuously fetches order book data from the DeepBook sandbox and updates the React UI.

## What you will build

A `useEffect` hook (lines 116-145 of `src/App.tsx`) that:

1. Defines an async `tick()` function that calls `deepbook.getLevel2TicksFromMid` via `withRetry`.
2. Fires `tick()` immediately on mount so the first data appears without waiting one full interval.
3. Schedules subsequent ticks with `setInterval` at the `poll_interval_ms` interval you configured in personalization.
4. Returns a cleanup function that cancels the interval and sets a `cancelled` flag to suppress stale state updates after unmount.
5. Handles errors gracefully — log them and update an error banner without crashing the component.

## Key concepts

- **Cancelled flag pattern**: React's `useEffect` cleanup fires when the component unmounts or the effect's dependency changes. Setting `cancelled = true` in the cleanup ensures that async callbacks from the previous render cycle do not call `setState` on an unmounted component.
- **withRetry integration**: Wrap every `deepbook.*` call in `withRetry(() => deepbook.call(...))` so transient network blips are handled automatically.

The simulate-verification gate sends a request to `http://localhost:9009/manifest` (status 200) to confirm the sandbox is still up after your code compiles and mounts. If the manifest endpoint is reachable, the gate passes and your cursor advances to the next phase.
