# Phase 2 — Resilient gRPC Simulation Calls

In this phase you add fault-tolerance to the DeepBook SDK calls that go over gRPC to the local sandbox. The sandbox simulates Sui transactions in-process and communicates via gRPC; transient failures (container restart, brief port unavailability) are expected during development.

## What you will build

A generic `withRetry<T>` helper (lines 103–114 of `src/App.tsx`) that wraps any async function and:

1. Calls the function immediately.
2. On failure waits `baseDelayMs * 2^attempt` milliseconds before the next attempt (exponential backoff).
3. Retries up to `retries` times (default 3) then re-throws the last error if all attempts fail.

## Why this matters

Without retry logic, a single momentary gRPC error shows an error state to the user and stops all polling. With `withRetry`, the SDK calls become self-healing: a hiccup in the sandbox container does not interrupt the learner's session. Phase 3 will call SDK methods through `withRetry` so the polling loop inherits this resilience automatically.

## Verification

The phase verification mode is `compile` — a clean `pnpm build` confirms the helper is syntactically valid TypeScript with the correct generic signature.
