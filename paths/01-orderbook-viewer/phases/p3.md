# Phase 3 — Polling Loop with Failure Tolerance

Welcome to Phase 3 of the Orderbook Viewer path. In this phase you will implement the polling loop that continuously fetches mid price and level-2 order book data from the DeepBook sandbox and displays it in the UI.

## What you are building

The order book viewer needs to display live data that refreshes at your chosen interval. Your task is to implement a `useEffect` hook that:

1. Sets up a `setInterval` to fire every `poll_interval_ms` milliseconds (your personalization value).
2. On each tick, calls `withRetry(() => client.deepbook.midPrice(poolId))` and `withRetry(() => client.deepbook.getLevel2TicksFromMid(poolId, depth))`.
3. Updates React state with the fresh data so the UI re-renders.
4. Catches tick-level errors, increments a failure counter, and continues polling without aborting the interval.
5. Returns a cleanup function that calls `clearInterval` when the component unmounts.

## Why this matters

A live order book viewer must tolerate failures gracefully. If a single tick error cancelled the interval, the UI would freeze until the user refreshed the page. By catching errors and continuing the loop, the viewer self-heals when the sandbox recovers.

The `poll_interval_ms` personalization value gives each student control over the refresh rate — from aggressive 1-second polling to battery-friendly 30-second intervals.

## Acceptance

Phase 3 is complete when the sandbox manifest endpoint at `http://localhost:9009/manifest` responds with a valid JSON manifest (simulated by `pnpm simulate`), confirming that the polling loop connects to the live sandbox and receives real data.
